# AiLang Quick Reference (v0.2.0)

AiLang is a Rust-implemented compiled language that transpiles to C99 and links via `clang -O2`. The source syntax is optimized for low token count when generated by an LLM; the binary runs within ~8% of hand-written C.

## Mental model

- Default integer type is **`i64`**. Unannotated function parameters and locals are `i64` unless context says otherwise.
- All language keywords are **2 characters** (`fn`, `lp`, `mt`, `rt`, `el`, `mu`, `en`, …). Do not substitute `for`/`while`/`match`/`return`/`else`/`let`/`enum`.
- Top-level statements **auto-wrap into `fn main()`**. Don't write `fn main` yourself unless you specifically need to.
- The trailing expression of a function body is the **return value** — no `rt` needed at the end.
- Single-statement bodies skip `{...}`. `if x>0 print(x)` is legal.
- Trailing `if` / `mt` expressions are returned implicitly: `fn classify(n) { if n<0 "neg" el if n==0 "zero" el "pos" }` works with no `rt`.

## Keywords (every one is exactly 2 chars)

| kw | meaning |
|----|---------|
| `fn` | function or lambda |
| `if` / `el` | if / else |
| `lp` | loop — unifies `for`-in **and** `while` |
| `rt` | return (only needed for early return) |
| `mt` | match |
| `st` | struct |
| `en` | enum / algebraic data type |
| `tr` | trait (parsed; runtime support minimal) |
| `im` | import |
| `ex` | extern (C ABI / FFI) |
| `mu` | mutable variable / mutable param marker |
| `br` / `ct` | break / continue |
| `as` | rename in `im` |
| `in` | iterator binding in `lp` |
| `ok` | construct an `!T` success value |
| `true` `false` `nil` | literals |

Type names (also keywords): `i8 i16 i32 i64 u8 u16 u32 u64 f32 f64 bool str bytes`.

`_` is the **wildcard pattern** (not a regular identifier).

## Operators

```
arithmetic  +  -  *  /  %         (+ also concatenates when an operand is str)
compound    +=  -=  *=  /=  %=
compare     ==  !=  <  <=  >  >=
logical     &&  ||  !
bitwise     &  |  ^  ~  <<  >>
string      ++                    (explicit concat — legacy; prefer +)
range       ..  ..=               (exclusive / inclusive)
declare     :=                    (NEW binding — use this to introduce a name)
assign      =                     (re-assign an existing mu binding)
return ty   ->
match arm   =>
pipe        |>                    (x |> f(a)  becomes  f(x, a))
nil coal    ??                    (a ?? b: a if non-nil else b)
ternary     cond ? a : b
optional    ?T                    (type prefix: nil allowed)
result      !T                    (type prefix: fallible — see Error handling)
try         expr?                 (postfix: propagate err to enclosing !T fn)
```

**Critical:** `:=` introduces a new name. `=` mutates an existing `mu`-binding. Mixing them up is the single most common mistake.

## Bindings

```
x := 10           // immutable binding
mu n := 5         // mutable — required to write `n = ...` later
n = 7             // ok because n is mu
n += 1            // compound assign also requires mu
```

## Functions and closures

Named function:
```
fn add(a, b) a + b                            // expression body
fn fib(n) {                                    // block body
  if n < 2 rt n                                // braceless if + early return
  fib(n-1) + fib(n-2)                          // trailing expr = return value
}
fn greet(name:str) "Hello, " + name + "!"      // annotate non-i64 params
```

Lambda (closure literal) — exact syntax is `fn(params) body`:
```
double  := fn(x) x * 2                         // closes over nothing
add     := fn(a, b) a + b
greeter := fn(name:str) "Hello, " + name + "!"

threshold := 3                                  // captured by value
gt := fn(x) x > threshold                       // gt(5) → true

println(double(7))                              // 14
```

Higher-order: pass lambdas to `map` / `filter` / `reduce` (provided by stdlib or the user):
```
nums := [1, 2, 3, 4, 5]
doubled := map(nums, fn(x) x * 2)               // [2,4,6,8,10]
odds    := filter(nums, fn(x) x % 2 == 1)
total   := reduce(nums, 0, fn(a, b) a + b)
```

**Closure gotcha — capture is by value at creation time.** Later mutation of the captured `mu` is invisible to the lambda. If you need the latest value, pass it as a parameter.

**Lambda syntax gotchas:** it is `fn(x) body`, NOT `|x| body` (Rust) or `(x) => body` (JS) or `lambda x: body` (Python).

Mutable parameter — receive a value, modify the local copy, return it:
```
fn shift(mu n, by) { n = n + by; n }
println(shift(10, 3))                           // 13
```

Inference rules (unchanged):
- Unannotated parameter → `i64`
- Return type inferred from the tail expression: `str` if it produces a string, `void` for a call to a void fn like `println`, else `i64`. For lambdas, the same rules apply.
- Annotate `:str` / `:bool` / `:[T]` / `:{K:V}` / `:Shape` (custom type) explicitly when the param isn't an i64.

## Control flow

```
// for-in (exclusive range; ..= is inclusive)
lp i in 1..10 {
  print(i); print(" ")
}

// while form — same lp keyword
mu n := 5
lp n > 0 {
  println(n)
  n -= 1
}

// if / el
if x > 0 {
  println("positive")
} el if x == 0 {
  println("zero")
} el {
  println("negative")
}

// match — arm separator is `;`
mt (i%3, i%5) {
  (0,0) => println("FizzBuzz");
  (0,_) => println("Fizz");
  (_,0) => println("Buzz");
  _     => println(i);
}

// map iteration with tuple destructure
lp (k, v) in m {
  print(k); print(":"); println(v)
}
```

## Algebraic data types (`en`)

Declare with `en Name { Variant, Variant(field:Type, ...), ... }`. Variants can be unit-only or carry typed payloads. Field type annotations are **required** in the declaration. Construct with bare `Variant` or `Variant(args)` — no namespace prefix. Destructure in `mt`:

```
en Shape {
  Circle(r:i64),
  Square(side:i64),
  Rect(w:i64, h:i64),
}

fn area(s:Shape) {
  mt s {
    Circle(r)  => r * r * 3;          // π ≈ 3
    Square(s)  => s * s;
    Rect(w, h) => w * h;
  }
}
println(area(Circle(5)))              // 75

en Color { Red, Green, Blue }         // classic enum (unit variants)
en Maybe { None, Some(v:i64) }        // Maybe-like
```

**Gotchas:**
- Variant constructors are **bare**, not namespaced: `Some(42)` and `None`, NOT `Maybe::Some(42)` or `Maybe.Some(42)`.
- Patterns bind **positionally**: `Circle(r)` works regardless of the field name in the decl. No `..` rest, no guards.
- Two enums in the same module **cannot share** a variant name (single global variant table).

## Generics

Type parameters in `<...>` after the fn name. The body is checked once; codegen monomorphizes one C function per concrete call-site type:

```
fn id<T>(x:T) -> T x
println(id(42))                       // 42    (i64 specialization)
println(id("hello"))                  // hello (str specialization)

fn first<T>(arr:[T]) -> T arr[0]
println(first([10, 20, 30]))          // 10
println(first(["alice", "bob"]))      // alice

fn pair<A, B>(a:A, b:B) -> A a        // multiple type params
```

`T` can appear bare, as `[T]`, or as `{K:V}`. Generics no longer collapse to `i64` — each call-site type gets its own specialization.

## Error handling: `!T` + `?` propagation

`!T` is a fallible result type. Constructors:
- `ok(v)` — polymorphic success
- `err_i64(msg)`, `err_str(msg)`, `err_bool(msg)`, `err_f64(msg)` — **type baked into the name**, one per primitive
- Inspectors: `unwrap(r)`, `is_ok(r)`, `is_err(r)`, `err_msg(r)`

Postfix `?` propagates the first error to the enclosing fn's `!T` return:

```
fn parse_int_safe(s:str) -> !i64 {
  if regex_match("^-?[0-9]+$", s) rt ok(str_to_int(s))
  err_i64("not a number: " + s)
}

fn parse_sum(a:str, b:str, c:str) -> !i64 {
  x := parse_int_safe(a)?              // propagates on err
  y := parse_int_safe(b)?
  z := parse_int_safe(c)?
  ok(x + y + z)
}

r := parse_sum("1", "2", "3")
if is_ok(r) println(unwrap(r)) el println(err_msg(r))
```

**Gotchas:**
- There is no generic `err("msg")` — sema cannot infer `T` from a `str` arg. Always use the type-suffixed form: `err_i64`, `err_str`, `err_bool`, `err_f64`.
- `unwrap` aborts on error. Use `is_ok(r)` first if you need to handle the error path.
- `?` only works inside a fn whose return type is `!T` for the same `T`. **Implicit `main` is not `!T`-typed** — wrap fallible code in `fn run() -> !i64 { … }` and call it from main.

## Structs (`st`)

```
st Point { x:i64; y:i64 }              // fields separated by ; or ,

p := Point { x: 3, y: 4 }
println(p.x)                            // 3
println(p)                              // Point { x: 3, y: 4 } (auto pretty-print)
```

## Arrays, maps, bytes

```
nums  := [10, 20, 30]                   // [i64]
words := ["alice", "bob"]               // [str]
println(len(nums))                      // 3
println(nums[2])                        // 30
lp x in nums { total += x }

// Maps — multiple instantiations now supported
m1 := {1: 10, 2: 20}                    // {i64:i64}
m2 := {"alice": 90, "bob": 75}          // {str:i64}
m3 := {"name": "Ada", "lang": "AiL"}    // {str:str}
m1[4] = 40                              // insert / set
println(has(m1, 4))                     // true
println(m1[99])                          // missing key → 0 (no exception)

// bytes — distinct from str, may contain \0
b := str_to_bytes("hello\0world")
println(len(b))                          // 11
println(b[0])                            // 104   (= 'h')
slice := bytes_slice(b, 6, 11)           // bytes "world"
println(bytes_to_str(slice))             // world
```

Supported instantiations today: `[i64]`, `[str]`, `[bytes]`, `{i64:i64}`, `{str:i64}`, `{str:str}`. Other map combos may or may not be implemented — check examples before assuming.

## Pipe operator and string concat

```
fn double(x) x * 2
fn square(x) x * x

println(5 |> double |> square)           // square(double(5)) = 100

msg := "Hello, " + "AiLang" + "!"        // + concatenates when an operand is str
println(len(msg))                        // 14
```

## FFI to libc

```
ex fn puts(s:str) -> i32
ex fn abs(n:i32) -> i32

puts("hello via libc puts!")
println(abs(-7))                         // 7
```

`ex fn` declares an extern with C ABI — no body, linked directly. The driver always links `-lm` and `-lgc`; for other libs the linker discovers them by name.

## Modules and stdlib

```
im "mathlib.ail"                         // user module — relative path
im "std/math.ail"                        // stdlib
```

Resolution: first relative to the importing file, then to cwd. Imported `fn main` is silently dropped. Available `std/*.ail` modules:

| module | what it provides |
|--------|------------------|
| `std/math.ail`  | libm wrappers (`sqrt`, `pow`, `sin`, `cos`, `tan`, `log`, `log2`, `log10`, `exp`, `floor`, `ceil`), plus `min`, `max`, `ipow` (integer power), `gcd`, libc `abs`/`rand`/`srand` |
| `std/str.ail`   | `eq(a, b)`, `parse_int(s)`, libc `strcmp`/`atoi` |
| `std/time.ail`  | `now_ms()`, `now_us()`, `mono_ms()`, `time_iso(ms)`, `now_iso()`, `tick()`/`since(t)` stopwatch, `sleep_ms`/`sleep_s` |
| `std/sock.ail`  | `tcp_listen(host, port)`, `tcp_accept(fd)`, `tcp_connect(host, port)`, `sock_send`, `sock_send_str_all(fd, s)`, `sock_recv(fd, max) -> bytes`, `sock_close`. Returns `-1` on error. |
| `std/http.ail`  | `http_text(status, body)`, `http_json(status, body)`, request parsers `http_method/http_path/http_header(req, name)/http_body/http_content_length`, `http_recv_request(fd)` (handles Content-Length). Keep-alive variants suffixed `_ka`. |
| `std/redis.ail` | `redis_connect(host, port)`, `redis_close`, `redis_ping`, `redis_set(fd, k, v)`, `redis_get(fd, k)`, `redis_incr(fd, k)`, `redis_del(fd, k)`, `redis_exists(fd, k)`. Returns `""` on nil/error. |

| `std/tls.ail`   | OpenSSL TLS. `tls_server_ctx(cert, key)`, `tls_client_ctx()`, `tls_accept(ctx, fd) -> ssl`, `tls_connect_fd(ctx, fd)`, `tls_send_str_all(ssl, s)`, `tls_recv(ssl, max) -> bytes`, `tls_close`, `tls_free_ctx`, `tls_error()` |

## Built-in functions

| Name | Signature | Notes |
|------|-----------|-------|
| `print(x)` | `(any) -> ()` | no newline; dispatched on arg type via C11 `_Generic` |
| `println(x)` | `(any) -> ()` | adds `"\n"`. Pretty-prints structs/enums automatically |
| `len(x)` | `(str \| bytes \| [T] \| {K:V}) -> i64` | byte length / element count |

| `unwrap(r)`, `is_ok(r)`, `is_err(r)`, `err_msg(r)` | inspectors for `!T` | `unwrap` aborts on err |

## Compile and run

```bash
ailangc run prog.ail                # compile + execute (forwards exit code)
ailangc compile prog.ail            # produces ./prog binary
ailangc compile prog.ail -o out     # custom output path
ailangc compile --emit-c prog.ail   # print generated C (debug aid)
ailangc parse  prog.ail             # dump AST
ailangc tokens prog.ail             # dump token stream
```

Output binary and generated `.c` land next to the source.

## What is NOT supported (don't generate these)

- Generic `err(msg)` constructor — use `err_i64` / `err_str` / `err_bool` / `err_f64`
- `?` postfix at top level (implicit-main scope) — wrap in `fn run() -> !T { … }`
- `match` guards (`if cond` on a pattern arm), rest patterns (`..`), or `@` bindings
- Closures that mutate captured variables (capture is by-value snapshot)
- `tr` trait method dispatch (the keyword parses but resolution is minimal)
- Long WebSocket frames (≥126 bytes) in `std/ws.ail` — current impl is short-text only
- Nested JSON via `std/json.ail` — only one-level objects

## Idiomatic vs non-idiomatic

Prefer:
- Implicit `main` for top-level scripts
- Expression-bodied fns: `fn add(a, b) a + b`
- Trailing-expression return (no `rt` at end of body)
- Trailing `if` / `mt` as return value
- Braceless single-statement bodies
- `+` for string concat (not `++`)
- `lp x in coll` over manual index counters
- `mt` over chained `if`/`el if` for value matching
- Bare variant names: `Some(42)`, NOT `Maybe::Some(42)`
- `fn(x) body` lambdas, NOT `|x| body` (Rust) or `(x) => body` (JS)

Avoid:
- `let`/`var`/`enum`/`return`/`for`/`while`/`else` — wrong keywords
- `: i64` and `-> i64` annotations when default already gives i64
- `Err("msg")` / `Result::Ok` — it's `ok(v)` and `err_<T>("msg")`
- `Maybe.Some(v)` — variants are global, bare

## Worked examples

### Hello world (implicit main)
```
println("hello, AiLang")
```

### FizzBuzz with tuple match
```
lp i in 1..16 {
  mt (i%3, i%5) {
    (0,0) => println("FizzBuzz");
    (0,_) => println("Fizz");
    (_,0) => println("Buzz");
    _     => println(i);
  }
}
```

### Recursive fib + iterative fib
```
fn fib(n) {
  if n < 2 rt n
  fib(n-1) + fib(n-2)
}
println(fib(30))                          // 832040

fn fib_iter(n) {
  if n < 2 rt n
  mu a := 0; mu b := 1; mu i := 2
  lp i <= n {
    t := a + b
    a = b; b = t
    i += 1
  }
  b
}
println(fib_iter(20))                     // 6765
```

### ADT + match (shapes)
```
en Shape {
  Circle(r:i64),
  Square(side:i64),
  Rect(w:i64, h:i64),
}

fn area(s:Shape) {
  mt s {
    Circle(r)  => r * r * 3;
    Square(s)  => s * s;
    Rect(w, h) => w * h;
  }
}

shapes := [Circle(5), Square(4), Rect(3, 6)]
lp s in shapes { println(area(s)) }
```

### Higher-order with closures
```
fn map_i64(arr:[i64], f) {
  mu out := [0]
  out[0] = f(arr[0])
  lp i in 1..len(arr) { out[i] = f(arr[i]) }   // sketch — real impl appends
  out
}

nums := [1, 2, 3, 4, 5]
threshold := 2
big := filter(nums, fn(x) x > threshold)       // [3,4,5]
sum := reduce(nums, 0, fn(a, b) a + b)         // 15
println(sum)
```

### Generics — id and first
```
fn id<T>(x:T) -> T x
fn first<T>(arr:[T]) -> T arr[0]

println(id(42))                                // 42
println(id("hello"))                           // hello
println(first([10, 20, 30]))                   // 10
println(first(["alice", "bob"]))               // alice
```

### Result / error propagation
```
im "std/str.ail"

fn parse_pair(a:str, b:str) -> !i64 {
  x := parse_int(a)?                            // propagates err
  y := parse_int(b)?
  ok(x + y)
}

fn run() -> !i64 {
  total := parse_pair("12", "30")?
  ok(total)
}

r := run()
if is_ok(r) println(unwrap(r)) el println(err_msg(r))
```

### Bubble sort (arrays + nested loop + swap)
```
mu arr := [5, 2, 8, 1, 9, 3, 7, 4, 6]
n := len(arr)
lp i in 0..n {
  lp j in 0..(n-i-1) {
    if arr[j] > arr[j+1] {
      t := arr[j]
      arr[j] = arr[j+1]
      arr[j+1] = t
    }
  }
}
lp x in arr { print(x); print(" ") }
println("")
```

### String map — word counter
```
words := ["alice", "bob", "alice", "carol", "bob", "alice"]
mu counts := {"_seed_": 0}                       // seed for type inference
lp w in words {
  counts[w] = counts[w] + 1
}
lp (k, v) in counts {
  if k != "_seed_" { print(k); print(":"); println(v) }
}
```

## Top gotchas an LLM coming from Rust/Python/Go gets wrong

1. **`err_<T>` is type-suffixed, not generic.** Write `err_i64("bad")`, NOT `err("bad")` or `Err("bad")`. `ok(v)` *is* polymorphic.
2. **`?` only works inside `-> !T` fns.** Implicit `main` is not `!T`-typed. Wrap in `fn run() -> !i64 { … }`.
3. **Lambda syntax is `fn(x) body`** — NOT `|x| body`, `(x) => body`, or `lambda x: body`. Capture is by value at creation; later mutation of the captured `mu` is invisible to the lambda.
4. **`mt` arm separator is `;`** (not `,`). Variant patterns bind positionally — `Circle(r)` doesn't have to match the field name in the decl.
5. **Variant constructors are bare**, not namespaced: `Some(42)` / `None`, not `Maybe::Some(42)` or `Maybe.Some(42)`. Two enums in the same module can't share a variant name.
6. **Real types in keywords**: it's `en`, not `enum`; `st`, not `struct`; `fn`, not `function`; `mt`, not `match`; `rt`, not `return`.

When in doubt, mimic the shape of programs in `examples/*.ail` rather than translating from another language.
