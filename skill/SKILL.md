# AiLang Quick Reference

AiLang is a Rust-implemented compiled language that transpiles to C99 and links via `clang -O2`. The source syntax is optimized for low token count when generated by an LLM; the binary runs within ~8% of hand-written C.

## Mental model

- Default integer type is **`i64`**. Unannotated function parameters and locals are `i64` unless context says otherwise.
- All keywords are **2 characters**. They are unusual (`fn`, `lp`, `mt`, `rt`, `el`, `mu`, …) — do not substitute `for`/`while`/`match`/`return`/`else`/`let`.
- Top-level statements **auto-wrap into `fn main()`**. Don't write `fn main` yourself unless you have a specific reason.
- The trailing expression of a function body is the **return value** — no `rt` needed at the end.
- Single-statement bodies skip `{...}`. `if x>0 print(x)` is legal.

## Keywords (every one is exactly 2 chars)

| kw | meaning |
|----|---------|
| `fn` | function definition |
| `if` / `el` | if / else |
| `lp` | loop — unifies `for`-in **and** `while` |
| `rt` | return (only needed for early return) |
| `mt` | match |
| `st` | struct |
| `tr` | trait |
| `im` | import |
| `ex` | extern (C ABI / FFI) |
| `mu` | mutable variable marker |
| `br` / `ct` | break / continue |
| `as` | rename in `im` |
| `in` | iterator binding in `lp` |
| `true` `false` `nil` | literals |

Type names (also keywords): `i8 i16 i32 i64 u8 u16 u32 u64 f32 f64 bool str`.

`_` is the **wildcard pattern** (not a regular identifier).

## Operators

```
arithmetic  +  -  *  /  %         (+ also concatenates when an operand is str)
compound    +=  -=  *=  /=  %=
compare     ==  !=  <  <=  >  >=
logical     &&  ||  !
bitwise     &  |  ^  ~  <<  >>
string      ++                    (explicit concat — legacy; prefer +)
range       ..  ..=               (exclusive / inclusive)
declare     :=                    (NEW binding — use this to introduce a name)
assign      =                     (re-assign an existing mu binding)
return ty   ->
match arm   =>
pipe        |>                    (x |> f(a)  becomes  f(x, a))
nil coal    ??                    (a ?? b: a if non-nil else b)
ternary     cond ? a : b
optional    ?T                    (type prefix: nil allowed)
result      !T                    (type prefix: error possible)
try         expr?                 (postfix: propagate error)
```

**Critical:** `:=` introduces a new name. `=` mutates an existing `mu`-binding. Mixing them up is the single most common mistake.

## Bindings

```
x := 10           // immutable binding (you cannot re-assign x)
mu n := 5         // mutable — required to write `n = ...` later
n = 7             // ok because n is mu
n += 1            // compound assign also requires mu
```

## Functions

```
fn add(a, b) a + b                            // expression body
fn fib(n) {                                    // block body
  if n < 2 rt n                                // braceless if + early return
  fib(n-1) + fib(n-2)                          // trailing expr = return value
}
fn greet(name:str) "Hello, " + name + "!"      // annotate when not i64
```

Inference rules:
- Unannotated parameter → `i64`
- Return type inferred from the tail expression: `str` if it produces a string, `void` if it's a call to a void fn like `println`, else `i64`
- Annotate string/bool/array/map parameters explicitly — inference does **not** look at call sites

## Control flow

```
// for-in (exclusive range; use ..= for inclusive)
lp i in 1..10 {
  print(i); print(" ")
}

// while form — same lp keyword
mu n := 5
lp n > 0 {
  println(n)
  n -= 1
}

// if / el
if x > 0 {
  println("positive")
} el if x == 0 {
  println("zero")
} el {
  println("negative")
}

// match — arm separator is `;`
mt (i%3, i%5) {
  (0,0) => println("FizzBuzz");
  (0,_) => println("Fizz");
  (_,0) => println("Buzz");
  _     => println(i);
}
```

## Arrays and maps

```
nums := [10, 20, 30]              // [i64]
words := ["alice", "bob"]         // [str]
println(len(nums))                // 3
println(nums[2])                  // 30
lp x in nums { total += x }

m := {1: 10, 2: 20, 3: 30}        // {i64:i64} — currently the only map type
m[4] = 40                          // insert / set
println(has(m, 4))                 // true
println(m[99])                     // missing key → 0 (no exception)
```

Supported instantiations today: `[i64]`, `[str]`, `{i64:i64}`. Other combos are not implemented.

## Pipe and string concat

```
fn double(x) x * 2
fn square(x) x * x

println(5 |> double |> square)    // 100

msg := "Hello, " + "AiLang" + "!"  // + concatenates when an operand is str
println(len(msg))                   // 14
```

## FFI to libc

```
ex fn puts(s:str) -> i32
ex fn abs(n:i32) -> i32

puts("hello via libc puts!")
println(abs(-7))                   // 7
```

`ex fn` declares an extern with C ABI — no body, linked directly.

## Modules

```
// in mathlib.ail
fn square(x) x * x

// in main.ail
im "mathlib.ail"
println(square(7))                 // 49
```

Import path is resolved relative to the importing file first, then to cwd. Imported `fn main` is silently dropped (only the root file's main runs).

A small stdlib lives in `std/`:
- `std/math.ail` — `min`, `max`, `pow`, `gcd`, libc `abs`/`rand`/`srand`
- `std/str.ail` — libc `strcmp`/`atoi` + `parse_int`

```
im "std/math.ail"
println(max(3, 7))                 // 7
```

## Built-in functions

| Name | Signature | Notes |
|------|-----------|-------|
| `print(x)` | `(any) -> ()` | no newline; dispatched on arg type via C11 `_Generic` |
| `println(x)` | `(any) -> ()` | adds `"\n"` |
| `len(x)` | `(str \| [T] \| {K:V}) -> i64` | byte length for str, element count otherwise |
| `has(m, k)` | `({K:V}, K) -> bool` | map membership |

## Compile and run

```bash
ailangc run prog.ail              # compile + execute (forwards exit code)
ailangc compile prog.ail          # produces ./prog binary
ailangc compile prog.ail -o out   # custom output path
ailangc compile --emit-c prog.ail # print generated C (debug aid)
ailangc parse  prog.ail           # dump AST
ailangc tokens prog.ail           # dump token stream
```

Output binary lands next to the source. Generated `.c` lands there too (gitignored under `examples/`).

## What is NOT supported (don't generate these)

- Lambdas / closures
- Traits with method dispatch (`tr` is parsed but no impl resolution)
- Generic functions that depend on the type variable for codegen — all type vars currently collapse to `i64`
- Maps other than `{i64:i64}`
- Sum types / enums beyond what `mt` patterns express
- `async` / coroutines / `Result` plumbing (the `?T`/`!T`/`expr?` syntax is parsed but runtime semantics are minimal)

## Idiomatic vs non-idiomatic

Prefer:
- Implicit `main` for top-level scripts — do **not** wrap in `fn main()` if the program is a few top-level calls
- Expression-bodied fns: `fn add(a, b) a + b`
- Trailing-expression return — no `rt` at end of body
- Braceless single-statement `if` / `lp` bodies
- `+` for string concat (not `++`)
- `lp x in coll` over manual index counters
- `mt` over chained `if`/`el if` when matching values

Avoid:
- `let`/`var` (the keyword is `:=`, optionally preceded by `mu`)
- `for`/`while` (it's `lp` for both)
- `return` at the end of a function body (let the trailing expression be the return)
- `: i64` and `-> i64` annotations when the default already gives you i64

## Worked examples — read these if unsure

### Hello world (implicit main)

```
// hello.ail
println("hello, AiLang")
```

### FizzBuzz with tuple match

```
// fizzbuzz.ail
lp i in 1..16 {
  mt (i%3, i%5) {
    (0,0) => println("FizzBuzz");
    (0,_) => println("Fizz");
    (_,0) => println("Buzz");
    _     => println(i);
  }
}
```

### Recursive fib (early return + trailing expr)

```
// fib.ail
fn fib(n) {
  if n < 2 rt n
  fib(n-1) + fib(n-2)
}
println(fib(30))   // 832040
```

### Strings + len (string param must be annotated)

```
// greet.ail
fn greet(name:str) "Hello, " + name + "!"
msg := greet("AiLang")
println(msg)        // Hello, AiLang!
println(len(msg))   // 14
```

### Sum of array (for-in + mutable accumulator)

```
// sum_of_array.ail
nums := [10, 20, 30, 40, 50]
mu total := 0
lp x in nums {
  total += x
}
println(total)   // 150
```

### Bubble sort (nested lp + array mutation + swap)

```
// bubble.ail
mu arr := [5, 2, 8, 1, 9, 3, 7, 4, 6]
n := len(arr)
lp i in 0..n {
  lp j in 0..(n-i-1) {
    if arr[j] > arr[j+1] {
      t := arr[j]                 // temp swap — no built-in tuple-assign
      arr[j] = arr[j+1]
      arr[j+1] = t
    }
  }
}
lp x in arr { print(x); print(" ") }
println("")                       // 1 2 3 4 5 6 7 8 9
```

### Number-frequency counter (map indexing, missing-key returns 0)

```
// frequency.ail
nums := [1, 2, 2, 3, 3, 3, 4, 4, 4, 4]
mu counts := {0:0}                // need {i64:i64} type; seed key is dummy
lp x in nums {
  counts[x] = counts[x] + 1        // missing key reads as 0, no exception
}
lp k in 1..5 {
  print(k); print(":"); println(counts[k])
}
// 1:1   2:2   3:3   4:4
```

### Pipe pipeline (`|>` becomes left-arg insertion)

```
// pipeline.ail
fn double(x) x * 2
fn add(a, b) a + b
fn square(x) x * x

// 5 |> double   = double(5)      = 10
// 10 |> add(7)  = add(10, 7)     = 17
// 17 |> square  = square(17)     = 289
println(5 |> double |> add(7) |> square)
```

### Multi-module program (im + stdlib usage)

```
// in mathlib.ail
fn area(w, h) w * h
fn perimeter(w, h) 2 * (w + h)
```

```
// in main.ail
im "mathlib.ail"
im "std/math.ail"           // pulls in min, max, pow, gcd, etc.

w := 7
h := 5
println(area(w, h))         // 35
println(perimeter(w, h))    // 24
println(max(w, h))          // 7
println(gcd(36, 24))        // 12
```

### Iterative fib (lp as while + mutable bindings)

```
// fib_iter.ail
fn fib(n) {
  if n < 2 rt n
  mu a := 0
  mu b := 1
  mu i := 2
  lp i <= n {
    t := a + b
    a = b
    b = t
    i += 1
  }
  b
}
println(fib(20))            // 6765
```

### FFI to libc — seed + random number

```
// random.ail
ex fn srand(seed:u32)
ex fn rand() -> i32

srand(42)
lp i in 0..5 {
  println(rand() % 100)
}
```

Working programs live under `examples/*.ail` in the AiLang repo. When in doubt, mimic their shape rather than translating literally from another language.
